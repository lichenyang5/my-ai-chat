# 生命周期与副作用

## 一、类组件生命周期

> 虽然新项目用函数组件，但面试经常问类组件生命周期，老项目也会遇到。

### 生命周期三个阶段

```
挂载（Mount）          更新（Update）           卸载（Unmount）
─────────────         ─────────────           ─────────────
constructor()         static getDerivedState   componentWillUnmount()
    ↓                   FromProps()                 ↓
static getDerived         ↓                   清理工作（取消订阅、
  StateFromProps()   shouldComponentUpdate()    清除定时器等）
    ↓                     ↓
render()              render()
    ↓                     ↓
componentDidMount()   componentDidUpdate()
```

### 常用生命周期

| 生命周期 | 阶段 | 作用 |
|---------|------|------|
| `constructor` | 挂载 | 初始化 state |
| `render` | 挂载/更新 | 返回 JSX（纯函数，不能有副作用） |
| `componentDidMount` | 挂载后 | 发请求、订阅事件、操作 DOM |
| `componentDidUpdate` | 更新后 | state/props 变化后执行逻辑 |
| `componentWillUnmount` | 卸载前 | 清理定时器、取消订阅 |
| `shouldComponentUpdate` | 更新前 | 返回 false 可阻止渲染（性能优化） |

### 类组件代码示例

```jsx
class UserPage extends React.Component {
  constructor(props) {
    super(props);
    this.state = { user: null, loading: true };
  }

  // 挂载后：发请求
  componentDidMount() {
    this.fetchUser(this.props.userId);
  }

  // 更新后：userId 变了重新请求
  componentDidUpdate(prevProps) {
    if (prevProps.userId !== this.props.userId) {
      this.fetchUser(this.props.userId);
    }
  }

  // 卸载前：清理
  componentWillUnmount() {
    // 取消请求、清除定时器等
  }

  fetchUser(id) {
    fetch(`/api/user/${id}`)
      .then(res => res.json())
      .then(user => this.setState({ user, loading: false }));
  }

  render() {
    if (this.state.loading) return <p>加载中...</p>;
    return <p>{this.state.user.name}</p>;
  }
}
```

## 二、函数组件用 useEffect 替代生命周期

| 类组件生命周期 | 函数组件 useEffect 等价写法 |
|--------------|--------------------------|
| `componentDidMount` | `useEffect(() => {}, [])` |
| `componentDidUpdate` | `useEffect(() => {}, [dep])` |
| `componentWillUnmount` | `useEffect(() => { return () => {} }, [])` |
| `shouldComponentUpdate` | `React.memo()` |

### 对照代码

```jsx
// 上面的类组件，用函数组件改写：
function UserPage({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // componentDidMount + componentDidUpdate
  useEffect(() => {
    setLoading(true);
    fetch(`/api/user/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });

    // componentWillUnmount（清理）
    return () => {
      // 取消请求等清理操作
    };
  }, [userId]);  // userId 变化时重新执行

  if (loading) return <p>加载中...</p>;
  return <p>{user.name}</p>;
}
```

## 三、useEffect 深入理解

### useEffect 的心智模型

不要把 useEffect 当"生命周期"来想，要当**"数据同步"**来想：

```
❌ 错误思维：挂载时请求、更新时请求、卸载时清理
✅ 正确思维：让副作用和 userId 保持同步，userId 变了就重新请求
```

### 常见副作用场景

```jsx
function App() {
  // ① 请求数据
  useEffect(() => {
    fetchData();
  }, []);

  // ② 订阅/监听事件
  useEffect(() => {
    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  // ③ 定时器
  useEffect(() => {
    const timer = setInterval(() => tick(), 1000);
    return () => clearInterval(timer);
  }, []);

  // ④ 修改 document.title
  useEffect(() => {
    document.title = `你有 ${count} 条消息`;
  }, [count]);

  // ⑤ 操作 DOM
  useEffect(() => {
    inputRef.current.focus();
  }, []);
}
```

### useEffect 依赖数组的陷阱

```jsx
// ❌ 遗漏依赖：effect 里用了 count 但没加到依赖中
useEffect(() => {
  const timer = setInterval(() => {
    setCount(count + 1);  // count 永远是初始值（闭包陷阱）
  }, 1000);
  return () => clearInterval(timer);
}, []);  // ← 缺少 count

// ✅ 方案一：加上依赖（但每次 count 变化都重新设定时器）
useEffect(() => { ... }, [count]);

// ✅ 方案二：函数式更新（推荐，不需要依赖 count）
useEffect(() => {
  const timer = setInterval(() => {
    setCount(prev => prev + 1);  // 不依赖外部 count
  }, 1000);
  return () => clearInterval(timer);
}, []);  // 空依赖没问题了
```

## 四、useLayoutEffect

和 useEffect 的区别：

| 对比 | useEffect | useLayoutEffect |
|------|-----------|-----------------|
| 执行时机 | 渲染到屏幕**之后**异步执行 | 渲染到屏幕**之前**同步执行 |
| 阻塞渲染 | 不阻塞 | 阻塞（同步） |
| 用途 | 绝大多数场景 | 需要在绘制前读/改 DOM |

```jsx
// 场景：防止页面闪烁
// 比如需要根据 DOM 尺寸计算位置，用 useEffect 会先渲染→再调整→闪一下
// 用 useLayoutEffect 在绘制前就算好，不闪烁

useLayoutEffect(() => {
  const { height } = ref.current.getBoundingClientRect();
  setPosition(height > 100 ? 'bottom' : 'top');
}, []);
```

> 经验：**99% 的情况用 useEffect**，只有明确需要在绘制前操作 DOM 时才用 useLayoutEffect。

## 五、高频面试题

### Q1：父子组件的生命周期执行顺序？

**挂载时**（由外向内渲染，由内向外完成）：
```
父 constructor → 父 render → 子 constructor → 子 render → 子 componentDidMount → 父 componentDidMount
```

**更新时**：
```
父 render → 子 render → 子 componentDidUpdate → 父 componentDidUpdate
```

**卸载时**：
```
父 componentWillUnmount → 子 componentWillUnmount
```

### Q2：useEffect 和 useLayoutEffect 怎么选？

- 默认用 `useEffect`（异步，不阻塞渲染，性能好）
- 如果用了 useEffect 出现**页面闪烁**（先渲染错误位置再跳到正确位置），改用 `useLayoutEffect`

### Q3：为什么 useEffect 里不能直接用 async？

```jsx
// ❌ 错误：useEffect 的回调不能是 async 函数
useEffect(async () => {
  const data = await fetchData();  // 报错
}, []);

// ✅ 正确：在里面定义 async 函数再调用
useEffect(() => {
  const loadData = async () => {
    const data = await fetchData();
    setData(data);
  };
  loadData();
}, []);
```

原因：useEffect 期望回调返回 `undefined` 或清理函数，而 async 函数返回的是 Promise。
