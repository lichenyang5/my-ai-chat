# 组件与 Props / State

## 一、React 组件的两种写法

### 函数组件（推荐）

```jsx
// 函数组件：一个返回 JSX 的函数，首字母必须大写
function Welcome(props) {
  return <h1>你好，{props.name}</h1>;
}

// 箭头函数写法
const Welcome = (props) => {
  return <h1>你好，{props.name}</h1>;
};

// 使用
<Welcome name="张三" />
```

### 类组件（老项目常见）

```jsx
import { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>你好，{this.props.name}</h1>;
  }
}
```

### 函数组件 vs 类组件

| 对比 | 函数组件 | 类组件 |
|------|---------|--------|
| 写法 | 普通函数 | class + render() |
| 状态管理 | `useState` Hook | `this.state` |
| 生命周期 | `useEffect` Hook | `componentDidMount` 等 |
| this 问题 | 没有 this | 要注意 this 指向 |
| 代码量 | 更少 | 更多 |
| 官方推荐 | ✅ 推荐 | 维护旧代码用 |

> React 16.8 之后有了 Hooks，函数组件能做类组件所有的事，新项目**一律用函数组件**。

## 二、Props（属性）

> 一句话：**父组件传给子组件的数据**，子组件只能读不能改（只读）。

### 基本用法

```jsx
// 父组件：传 props
function App() {
  return (
    <UserCard
      name="张三"
      age={25}
      isAdmin={true}
      hobbies={['篮球', '编程']}
      onDelete={() => console.log('删除')}
    />
  );
}

// 子组件：接收 props（解构写法更清晰）
function UserCard({ name, age, isAdmin, hobbies, onDelete }) {
  return (
    <div>
      <h2>{name}，{age}岁</h2>
      {isAdmin && <span>管理员</span>}
      <ul>
        {hobbies.map((h, i) => <li key={i}>{h}</li>)}
      </ul>
      <button onClick={onDelete}>删除</button>
    </div>
  );
}
```

### Props 能传什么？

| 类型 | 示例 |
|------|------|
| 字符串 | `name="张三"` |
| 数字 | `age={25}` |
| 布尔 | `isAdmin={true}` 或直接写 `isAdmin` |
| 数组/对象 | `list={[1,2,3]}` |
| 函数 | `onClick={() => {}}` |
| JSX / 组件 | `icon={<Icon />}` |
| children | 标签之间的内容 |

### children 属性

```jsx
// 父组件
<Card>
  <h2>标题</h2>
  <p>内容可以是任何东西</p>
</Card>

// 子组件：通过 props.children 拿到标签之间的内容
function Card({ children }) {
  return <div className="card">{children}</div>;
}
```

### 默认值

```jsx
function Button({ text = '点击', color = 'blue' }) {
  return <button style={{ color }}>{text}</button>;
}

// 不传 text 和 color 就用默认值
<Button />           // → 蓝色的"点击"按钮
<Button text="提交" /> // → 蓝色的"提交"按钮
```

## 三、State（状态）

> 一句话：**组件内部自己管理的数据**，state 变了界面自动重新渲染。

### Props vs State

| 对比 | Props | State |
|------|-------|-------|
| 谁控制 | 父组件传入 | 组件自己管理 |
| 可否修改 | ❌ 只读 | ✅ 通过 setState 修改 |
| 用途 | 父子通信 | 组件内部数据 |
| 变化后 | 子组件重新渲染 | 当前组件重新渲染 |

### useState 基本用法

```jsx
import { useState } from 'react';

function Counter() {
  // 解构：[当前值, 修改函数] = useState(初始值)
  const [count, setCount] = useState(0);
  const [name, setName] = useState('张三');
  const [list, setList] = useState([]);

  return (
    <div>
      <p>计数：{count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <button onClick={() => setCount(prev => prev + 1)}>+1（函数式）</button>
      <button onClick={() => setCount(0)}>重置</button>
    </div>
  );
}
```

### State 更新注意事项

```jsx
// ❌ 直接修改 state（不会触发重新渲染！）
count = count + 1;           // 错
list.push('新项');            // 错
user.name = '李四';           // 错

// ✅ 用 set 函数创建新值
setCount(count + 1);                        // 对
setList([...list, '新项']);                   // 对（展开旧数组 + 新项）
setUser({ ...user, name: '李四' });          // 对（展开旧对象 + 覆盖属性）
```

### 对象和数组的更新方式

```jsx
const [user, setUser] = useState({ name: '张三', age: 25 });
const [list, setList] = useState(['苹果', '香蕉']);

// 修改对象的某个字段
setUser({ ...user, age: 26 });

// 数组添加
setList([...list, '橘子']);

// 数组删除（过滤掉不要的）
setList(list.filter(item => item !== '香蕉'));

// 数组修改某一项
setList(list.map(item => item === '苹果' ? '大苹果' : item));
```

> 核心原则：**永远不要直接修改 state，要创建新的值**。React 靠比较引用（===）来判断是否变化。

## 四、组件通信

### 1. 父 → 子：Props

```jsx
// 父传子：最常见，通过 props
function Parent() {
  const [msg, setMsg] = useState('hello');
  return <Child message={msg} />;
}

function Child({ message }) {
  return <p>{message}</p>;
}
```

### 2. 子 → 父：回调函数

```jsx
// 子传父：父组件把函数传下去，子组件调用这个函数把数据带回来
function Parent() {
  const [childData, setChildData] = useState('');

  return (
    <div>
      <p>子组件传来的：{childData}</p>
      <Child onSend={(data) => setChildData(data)} />
    </div>
  );
}

function Child({ onSend }) {
  return <button onClick={() => onSend('我是子组件的数据')}>发送给父组件</button>;
}
```

### 3. 兄弟组件：状态提升

```jsx
// 兄弟通信：把共享状态提升到它们共同的父组件
function Parent() {
  const [data, setData] = useState('');

  return (
    <>
      <BrotherA onSend={setData} />
      <BrotherB data={data} />
    </>
  );
}

function BrotherA({ onSend }) {
  return <button onClick={() => onSend('来自A')}>发送</button>;
}

function BrotherB({ data }) {
  return <p>收到：{data}</p>;
}
```

### 4. 跨层级：Context

```jsx
import { createContext, useContext, useState } from 'react';

// 1. 创建 Context
const ThemeContext = createContext();

// 2. 顶层提供数据
function App() {
  const [theme, setTheme] = useState('light');

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Page />
    </ThemeContext.Provider>
  );
}

// 3. 任意深层子组件直接拿数据（不需要层层传 props）
function DeepChild() {
  const { theme, setTheme } = useContext(ThemeContext);

  return (
    <div>
      <p>当前主题：{theme}</p>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        切换主题
      </button>
    </div>
  );
}
```

### 通信方式总结

| 场景 | 方式 |
|------|------|
| 父 → 子 | Props |
| 子 → 父 | 回调函数 |
| 兄弟 | 状态提升到父组件 |
| 跨多层 | Context |
| 全局复杂状态 | Redux / Zustand |

## 五、高频面试题

### Q1：Props 能不能修改？

**不能**。Props 是只读的，子组件不能修改父组件传来的 props。如果需要"修改"，应该通过回调函数通知父组件去改。

### Q2：setState 是同步还是异步的？

在 React 18 中，**setState 全部是异步的**（自动批处理）：

```jsx
const handleClick = () => {
  setCount(1);
  setName('李四');
  // 这两次更新会合并成一次渲染，而不是渲染两次
};
```

如果需要拿到更新后的值，用**函数式更新**：
```jsx
setCount(prev => prev + 1);  // prev 一定是最新值
```

### Q3：为什么不能直接修改 state？

React 通过 `Object.is()` 比较前后 state 的**引用**来决定是否重新渲染：
- `setCount(newValue)` → 新引用 → React 知道变了 → 重新渲染
- `count = newValue` → 引用没变 → React 不知道 → 不渲染

### Q4：受控组件 vs 非受控组件？

| 对比 | 受控组件 | 非受控组件 |
|------|---------|-----------|
| 数据来源 | React state 控制 | DOM 自己管理 |
| 取值方式 | 从 state 读取 | 用 ref 读取 DOM |
| 推荐程度 | ✅ 推荐 | 特殊场景用 |

```jsx
// 受控组件：value + onChange 由 React 控制
const [val, setVal] = useState('');
<input value={val} onChange={e => setVal(e.target.value)} />

// 非受控组件：用 ref 直接读 DOM 值
const inputRef = useRef();
<input ref={inputRef} />
// 读取：inputRef.current.value
```
