# 事件机制与 State 更新

## 一、React 事件机制

### 基本用法

```jsx
function App() {
  // 事件处理函数
  const handleClick = (e) => {
    console.log('点击了按钮');
    console.log(e);  // React 合成事件对象（不是原生事件）
  };

  return (
    <div>
      {/* ① 用驼峰命名：onClick，不是 onclick */}
      {/* ② 传函数引用，不是函数调用 */}
      <button onClick={handleClick}>点击</button>

      {/* ③ 需要传参时，用箭头函数包一层 */}
      <button onClick={() => handleClick('参数')}>传参</button>

      {/* ❌ 常见错误：直接调用（组件渲染时就执行了） */}
      {/* <button onClick={handleClick('参数')}>错误</button> */}
    </div>
  );
}
```

### 合成事件（SyntheticEvent）

React 的事件不是直接绑定在 DOM 上的，而是用**合成事件**统一管理。

```
原生事件：直接绑在 DOM 元素上
React 事件：委托到 root 根节点上统一处理（事件委托）
```

```
点击 <button>
    ↓ 事件冒泡到根节点
React 根节点捕获
    ↓ 根据事件目标找到对应组件
    ↓ 创建 SyntheticEvent（合成事件）
    ↓ 执行对应的 onClick 处理函数
```

**为什么要用合成事件？**

| 好处 | 说明 |
|------|------|
| **跨浏览器兼容** | 抹平不同浏览器事件 API 的差异 |
| **性能优化** | 事件委托，不用给每个 DOM 都绑事件 |
| **统一管理** | 方便 React 内部做批量更新等优化 |

### 合成事件 vs 原生事件

| 对比 | React 合成事件 | 原生事件 |
|------|---------------|---------|
| 命名 | 驼峰（onClick） | 小写（onclick） |
| 绑定位置 | 委托到根节点 | 绑在具体 DOM |
| 事件对象 | SyntheticEvent | Event |
| 阻止默认 | `e.preventDefault()` | `return false` 也行 |
| 阻止冒泡 | `e.stopPropagation()` | 同 |

```jsx
function Form() {
  const handleSubmit = (e) => {
    e.preventDefault();   // 阻止表单默认提交
    console.log('表单提交');
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit">提交</button>
    </form>
  );
}
```

## 二、State 更新机制

### 批量更新（Batching）

React 18 中，**所有 setState 都会自动批量处理**：

```jsx
function App() {
  const [count, setCount] = useState(0);
  const [name, setName] = useState('张三');

  const handleClick = () => {
    setCount(1);       // 不会立即渲染
    setName('李四');    // 不会立即渲染
    // → React 把这两次更新合并，只渲染一次 ✅
  };

  // 即使在 setTimeout、Promise 里也是批量更新（React 18 新特性）
  const handleAsync = () => {
    setTimeout(() => {
      setCount(1);     // React 18：依然批量
      setName('李四');  // 只渲染一次
    }, 0);
  };

  console.log('渲染了');  // 只输出一次

  return <button onClick={handleClick}>{count} {name}</button>;
}
```

### 如果需要强制同步更新？

```jsx
import { flushSync } from 'react-dom';

const handleClick = () => {
  flushSync(() => {
    setCount(1);  // 立即渲染一次
  });
  // 这里 DOM 已经更新了
  flushSync(() => {
    setName('李四');  // 再渲染一次
  });
};
// 一共渲染 2 次（一般不需要这样做）
```

### 函数式更新 vs 直接赋值

```jsx
const [count, setCount] = useState(0);

// 场景：连续加 3 次
const handleClick = () => {
  // ❌ 直接赋值：三次都拿的是同一个 count（闭包陷阱），结果只 +1
  setCount(count + 1);  // 0 + 1 = 1
  setCount(count + 1);  // 0 + 1 = 1 （count 还是0）
  setCount(count + 1);  // 0 + 1 = 1 （count 还是0）
  // 最终：count = 1 ❌

  // ✅ 函数式更新：每次拿到的 prev 都是最新值
  setCount(prev => prev + 1);  // 0 → 1
  setCount(prev => prev + 1);  // 1 → 2
  setCount(prev => prev + 1);  // 2 → 3
  // 最终：count = 3 ✅
};
```

> 经验：**基于前一个值计算时，永远用函数式更新**。

## 三、闭包陷阱

这是 React Hooks 面试的**经典坑题**。

```jsx
function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      // ❌ 闭包陷阱：count 永远是 0（定义时的值）
      console.log(count);       // 永远输出 0
      setCount(count + 1);      // 永远是 0 + 1 = 1
    }, 1000);

    return () => clearInterval(timer);
  }, []);  // 空依赖 → effect 只执行一次 → 闭包锁死了 count = 0

  return <p>{count}</p>;  // 永远显示 1
}
```

**解决方案**：

```jsx
// 方案一：函数式更新（最简单）
setCount(prev => prev + 1);  // prev 总是最新值

// 方案二：用 useRef 保存最新值
const countRef = useRef(count);
countRef.current = count;  // 每次渲染同步更新

useEffect(() => {
  const timer = setInterval(() => {
    console.log(countRef.current);  // 总是最新值
  }, 1000);
  return () => clearInterval(timer);
}, []);
```

## 四、高频面试题

### Q1：React 为什么要用合成事件？

1. **跨浏览器兼容** — 统一不同浏览器的事件差异
2. **事件委托优化性能** — 所有事件委托到根节点，减少事件监听器数量
3. **配合 React 内部机制** — 方便实现批量更新、事件优先级调度

### Q2：setState 是同步还是异步？

React 18 中**默认都是异步的**（自动批量更新），不管在事件处理、setTimeout、Promise 中都是批量的。原因是为了性能优化，减少不必要的渲染次数。

### Q3：setState 后想立即拿到最新值怎么办？

```jsx
// 方法一：useEffect 监听变化
useEffect(() => {
  console.log('最新值:', count);
}, [count]);

// 方法二：flushSync 强制同步（不推荐）
flushSync(() => setCount(1));
// 此处 DOM 已更新

// 方法三：函数式更新中可以拿到最新值
setCount(prev => {
  console.log('最新值:', prev + 1);
  return prev + 1;
});
```

### Q4：什么是闭包陷阱？怎么解决？

在 useEffect/useCallback 中，由于闭包特性，回调函数会"记住"定义时的 state 值，后续 state 更新了但回调里还是旧值。

解决：
- 用**函数式更新** `setState(prev => ...)`
- 用 **useRef** 保存最新值
- 把依赖加到 **依赖数组** 中
