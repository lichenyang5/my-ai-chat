# React 基础与 JSX

## 一、React 是什么？

React 是 Facebook 开源的一个用于**构建用户界面**的 JavaScript 库。

> 一句话：用组件化的方式写 UI，数据变了界面自动更新。

**核心特点**：
- **组件化**：把页面拆成一个个独立的组件，像搭积木
- **声明式**：你描述"界面长什么样"，React 帮你操作 DOM
- **单向数据流**：数据从父组件流向子组件（props），不能反着来
- **虚拟 DOM**：先在内存中算好差异，再最小化更新真实 DOM

### React vs Vue 对比（面试常问）

| 对比项 | React | Vue |
|--------|-------|-----|
| 定位 | UI 库（只管视图层） | 渐进式框架（更全面） |
| 模板语法 | JSX（JS 里写 HTML） | template（HTML 里写指令） |
| 数据绑定 | 单向数据流 | 双向绑定（v-model） |
| 状态管理 | setState / Hooks | 响应式（ref / reactive） |
| 学习曲线 | 偏高（更灵活） | 偏低（约定更多） |
| 生态 | 社区驱动，选择多 | 官方配套，开箱即用 |
| 适用场景 | 大型复杂应用 | 中小型项目、快速开发 |

## 二、JSX 是什么？

JSX = **JavaScript XML**，让你在 JS 里写类似 HTML 的语法。

> 一句话：长得像 HTML，本质是 JS。

```jsx
// 这是 JSX
const element = <h1 className="title">Hello React</h1>;

// Babel 会把它编译成：
const element = React.createElement('h1', { className: 'title' }, 'Hello React');

// 最终生成虚拟 DOM 对象：
{
  type: 'h1',
  props: {
    className: 'title',
    children: 'Hello React'
  }
}
```

### JSX 语法规则

```jsx
function App() {
  const name = '张三';
  const isLogin = true;
  const list = ['苹果', '香蕉', '橘子'];

  return (
    // ① 必须有一个根元素（或用 <> 空标签包裹）
    <div>
      {/* ② 用 {} 嵌入 JS 表达式 */}
      <h1>你好，{name}</h1>

      {/* ③ class 要写成 className，for 要写成 htmlFor */}
      <label className="label" htmlFor="input">输入：</label>

      {/* ④ style 要写成对象形式，属性名用驼峰 */}
      <p style={{ color: 'red', fontSize: '16px' }}>红色文字</p>

      {/* ⑤ 条件渲染 */}
      {isLogin ? <p>已登录</p> : <p>请登录</p>}
      {isLogin && <p>欢迎回来</p>}

      {/* ⑥ 列表渲染（必须加 key） */}
      <ul>
        {list.map((item, index) => (
          <li key={index}>{item}</li>
        ))}
      </ul>

      {/* ⑦ 所有标签必须闭合 */}
      <img src="logo.png" alt="logo" />
      <br />
    </div>
  );
}
```

### JSX 和 HTML 的区别

| 区别 | JSX | HTML |
|------|-----|------|
| 属性名 | `className`、`htmlFor` | `class`、`for` |
| 样式 | `style={{ color: 'red' }}`（对象） | `style="color: red"`（字符串） |
| 事件 | `onClick`（驼峰） | `onclick`（小写） |
| 标签闭合 | 必须闭合 `<br />` | 可以不闭合 `<br>` |
| 根元素 | 必须有一个根元素 | 无限制 |
| 变量嵌入 | `{变量}` | 不支持 |

## 四、高频面试题

### Q1：JSX 是必须的吗？

不是必须的。JSX 只是 `React.createElement()` 的语法糖，没有 JSX 也能写 React，只是写起来很麻烦：

```jsx
// 用 JSX（简洁）
<div><h1>Hello</h1></div>

// 不用 JSX（繁琐）
React.createElement('div', null, React.createElement('h1', null, 'Hello'))
```

### Q2：React 为什么要用 className 而不是 class？

因为 JSX 本质是 JS，`class` 是 JS 的**保留关键字**（用于声明类），所以用 `className` 代替。

### Q3：为什么列表渲染必须加 key？

key 帮助 React 在 Diff 时**识别哪些元素变了、哪些没变**：
- 有 key → React 能精准找到变化的元素，只更新它
- 没 key / 用 index 做 key → React 可能搞混元素，导致错误复用和性能浪费

```jsx
// ❌ 不推荐：用 index 作为 key（列表会增删排序时出问题）
list.map((item, index) => <li key={index}>{item}</li>)

// ✅ 推荐：用唯一标识作为 key
list.map((item) => <li key={item.id}>{item.name}</li>)
```

### Q4：React 和 Vue 怎么选？

- **大型团队/复杂项目** → React（更灵活，TypeScript 支持好）
- **中小项目/快速开发** → Vue（上手快，生态统一）
- 两者都能胜任绝大多数场景，核心区别是**设计理念**：React 更像"库"（给你自由），Vue 更像"框架"（帮你约定）
