# React Router 与状态管理

## 一、React Router

### 是什么？

React 本身只管 UI，不管路由。**React Router** 是最主流的路由库，让单页应用（SPA）实现"多页面"效果。

> 一句话：URL 变了 → 渲染不同的组件，但页面不会刷新。

### 安装

```bash
npm install react-router-dom
```

### 基本用法（v6）

```jsx
import { BrowserRouter, Routes, Route, Link, useNavigate } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      {/* 导航链接 */}
      <nav>
        <Link to="/">首页</Link>
        <Link to="/about">关于</Link>
        <Link to="/user/123">用户</Link>
      </nav>

      {/* 路由配置 */}
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/user/:id" element={<User />} />
        <Route path="*" element={<NotFound />} />  {/* 404 */}
      </Routes>
    </BrowserRouter>
  );
}
```

### 常用 API

```jsx
import {
  useParams,       // 获取路由参数
  useSearchParams, // 获取查询参数
  useNavigate,     // 编程式导航
  useLocation,     // 获取当前路径信息
  Outlet           // 嵌套路由出口
} from 'react-router-dom';

// 获取动态参数：/user/:id
function User() {
  const { id } = useParams();  // { id: '123' }
  return <p>用户ID：{id}</p>;
}

// 获取查询参数：/search?keyword=react
function Search() {
  const [searchParams] = useSearchParams();
  const keyword = searchParams.get('keyword');  // 'react'
  return <p>搜索：{keyword}</p>;
}

// 编程式导航（代码跳转）
function LoginButton() {
  const navigate = useNavigate();

  const handleLogin = () => {
    // 登录成功后跳转
    navigate('/dashboard');          // 跳转
    navigate('/dashboard', { replace: true }); // 替换（不留历史记录）
    navigate(-1);                    // 后退
  };

  return <button onClick={handleLogin}>登录</button>;
}
```

### 嵌套路由

```jsx
function App() {
  return (
    <Routes>
      <Route path="/dashboard" element={<Dashboard />}>
        {/* 嵌套路由 */}
        <Route index element={<Overview />} />           {/* /dashboard */}
        <Route path="settings" element={<Settings />} />  {/* /dashboard/settings */}
        <Route path="profile" element={<Profile />} />    {/* /dashboard/profile */}
      </Route>
    </Routes>
  );
}

function Dashboard() {
  return (
    <div>
      <h1>控制台</h1>
      <nav>
        <Link to="/dashboard">概览</Link>
        <Link to="/dashboard/settings">设置</Link>
      </nav>
      {/* Outlet 渲染子路由匹配的组件 */}
      <Outlet />
    </div>
  );
}
```

### 路由守卫（权限控制）

```jsx
// React Router 没有内置路由守卫，自己封装一个
function PrivateRoute({ children }) {
  const isLogin = localStorage.getItem('token');

  if (!isLogin) {
    return <Navigate to="/login" replace />;  // 未登录 → 跳转登录页
  }

  return children;
}

// 使用
<Route
  path="/dashboard"
  element={
    <PrivateRoute>
      <Dashboard />
    </PrivateRoute>
  }
/>
```

### Hash 路由 vs History 路由

| 对比 | HashRouter | BrowserRouter |
|------|-----------|---------------|
| URL 样式 | `/#/about` | `/about` |
| 原理 | `window.onhashchange` | `history.pushState` |
| 服务器配置 | 不需要 | 需要配置回退到 index.html |
| SEO | 差 | 好 |
| 推荐 | 老项目/静态部署 | ✅ 新项目推荐 |

## 二、状态管理

### 什么时候需要状态管理？

```
组件自己的状态 → useState 就够了
父子共享 → props + 回调
跨多层共享 → Context
全局复杂状态 → Redux / Zustand
```

### 方案一：Context（轻量级）

适合：主题、语言、用户信息等**不频繁变化**的全局数据。

```jsx
// 完整示例见 03.md 的 useContext 部分
const ThemeContext = createContext();

// Provider 提供数据
<ThemeContext.Provider value={{ theme, setTheme }}>
  <App />
</ThemeContext.Provider>

// 任意子组件消费
const { theme } = useContext(ThemeContext);
```

**Context 的缺点**：value 变化时，所有消费该 Context 的组件都会重新渲染，即使它只用了其中一个字段。

### 方案二：Redux（重量级）

适合：大型应用、状态逻辑复杂、需要中间件（异步、日志）。

```
核心概念：
Store（仓库）→ 存全局状态
Action（动作）→ 描述"发生了什么"
Reducer（处理器）→ 根据 Action 返回新 State
Dispatch（派发）→ 触发 Action
```

```
用户点击 → dispatch(action) → reducer 处理 → 返回新 state → 组件更新
```

#### Redux Toolkit（现代写法，推荐）

```bash
npm install @reduxjs/toolkit react-redux
```

```jsx
// store/counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value += 1 },       // 可以直接"修改"（内部用 Immer）
    decrement: (state) => { state.value -= 1 },
    addBy: (state, action) => { state.value += action.payload },
  }
});

export const { increment, decrement, addBy } = counterSlice.actions;
export default counterSlice.reducer;
```

```jsx
// store/index.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './counterSlice';

export const store = configureStore({
  reducer: { counter: counterReducer }
});
```

```jsx
// main.jsx — 注入 Store
import { Provider } from 'react-redux';
import { store } from './store';

<Provider store={store}>
  <App />
</Provider>
```

```jsx
// 组件中使用
import { useSelector, useDispatch } from 'react-redux';
import { increment, addBy } from './store/counterSlice';

function Counter() {
  const count = useSelector(state => state.counter.value);  // 读
  const dispatch = useDispatch();                            // 写

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => dispatch(increment())}>+1</button>
      <button onClick={() => dispatch(addBy(5))}>+5</button>
    </div>
  );
}
```

### 方案三：Zustand（轻量推荐）

比 Redux 简单很多，没有 Action/Reducer/Provider 的概念。

```bash
npm install zustand
```

```jsx
import { create } from 'zustand';

// 创建 store（一个函数搞定）
const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
  reset: () => set({ count: 0 }),
}));

// 组件中使用（不需要 Provider）
function Counter() {
  const { count, increment, reset } = useCounterStore();

  return (
    <div>
      <p>{count}</p>
      <button onClick={increment}>+1</button>
      <button onClick={reset}>重置</button>
    </div>
  );
}
```

### 状态管理方案对比

| 方案 | 复杂度 | 适用场景 | 特点 |
|------|--------|---------|------|
| **useState** | 低 | 组件内部状态 | 最基础 |
| **Context** | 低 | 跨层级、不频繁变化 | 内置，无需安装 |
| **Redux Toolkit** | 中 | 大型应用、复杂状态 | 生态完善、中间件丰富 |
| **Zustand** | 低 | 中小型应用 | 极简、不需要 Provider |
| **Jotai/Recoil** | 低 | 原子化状态管理 | 细粒度更新 |

## 三、高频面试题

### Q1：React Router v5 和 v6 的区别？

| 对比 | v5 | v6 |
|------|----|----|
| 路由组件 | `<Switch>` | `<Routes>` |
| 路由定义 | `<Route component={Home}>` | `<Route element={<Home />}>` |
| 嵌套路由 | 手动嵌套 | 声明式嵌套 + `<Outlet>` |
| 导航 | `useHistory()` | `useNavigate()` |
| 重定向 | `<Redirect>` | `<Navigate>` |

### Q2：Redux 的工作流程？

```
用户操作 → dispatch(action) → reducer(state, action) → 返回新 state → store 更新 → 组件重新渲染
```

### Q3：什么时候用 Redux，什么时候用 Context？

- **Context**：全局主题、语言、用户信息等简单且不频繁变化的数据
- **Redux**：购物车、表单状态、复杂的异步数据流、需要时间旅行调试
- **Zustand**：想要 Redux 的能力但不想要 Redux 的复杂度
