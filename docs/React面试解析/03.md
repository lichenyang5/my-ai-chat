# Hooks 全面解析

## 一、Hooks 是什么？

> 一句话：让函数组件也能用 state、生命周期等能力的**特殊函数**，以 `use` 开头。

React 16.8 引入 Hooks 之前，函数组件只能展示 UI，不能有状态。有了 Hooks 之后，**函数组件可以完全替代类组件**。

### Hooks 使用规则（必须遵守）

1. **只能在函数组件或自定义 Hook 中使用**，不能在普通函数、类组件里用
2. **只能在顶层调用**，不能放在 if、for、嵌套函数里

```jsx
// ❌ 错误：条件语句中使用
if (isLogin) {
  const [name, setName] = useState('');  // 报错！
}

// ✅ 正确：顶层使用，条件逻辑放里面
const [name, setName] = useState('');
if (isLogin) {
  // 在这里用 name
}
```

## 二、常用 Hooks 详解

### 1. useState — 状态管理

```jsx
const [state, setState] = useState(初始值);
```

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  // 直接设置新值
  const add = () => setCount(count + 1);

  // 函数式更新（推荐：基于前一个值计算时用这个）
  const addSafe = () => setCount(prev => prev + 1);

  // 惰性初始化（初始值计算复杂时，只在首次渲染执行）
  const [data, setData] = useState(() => {
    return expensiveComputation();  // 只算一次
  });

  return <button onClick={addSafe}>计数：{count}</button>;
}
```

### 2. useEffect — 副作用处理

> 处理"跟渲染无关的事"：请求数据、订阅事件、操作 DOM、设置定时器...

```jsx
import { useState, useEffect } from 'react';

function UserPage({ userId }) {

  // ① 每次渲染后都执行（不传依赖数组）
  useEffect(() => {
    console.log('每次渲染都执行');
  });

  // ② 只在首次挂载时执行（空数组）
  useEffect(() => {
    console.log('只执行一次，相当于 componentDidMount');
  }, []);

  // ③ 依赖变化时执行
  useEffect(() => {
    console.log('userId 变了，重新请求');
    fetchUser(userId);
  }, [userId]);

  // ④ 清理函数（组件卸载或依赖变化前执行）
  useEffect(() => {
    const timer = setInterval(() => console.log('tick'), 1000);

    // 返回的函数 = 清理函数
    return () => {
      clearInterval(timer);  // 组件卸载时清除定时器
    };
  }, []);

  return <div>用户页面</div>;
}
```

**useEffect 执行时机**：

| 依赖数组 | 执行时机 |
|---------|---------|
| 不传 | 每次渲染后 |
| `[]` | 只在挂载时执行一次 |
| `[a, b]` | a 或 b 变化时执行 |
| 返回函数 | 卸载时 / 下次执行前 清理 |

### 3. useRef — 引用

> 两个用途：① 获取 DOM 元素 ② 保存不会触发渲染的变量

```jsx
import { useRef, useEffect } from 'react';

function TextInput() {
  // 用途一：获取 DOM
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();  // 自动聚焦
  }, []);

  // 用途二：保存变量（修改不会触发重新渲染）
  const timerRef = useRef(null);

  const start = () => {
    timerRef.current = setInterval(() => console.log('运行中'), 1000);
  };
  const stop = () => {
    clearInterval(timerRef.current);
  };

  return <input ref={inputRef} />;
}
```

**useRef vs useState**：

| 对比 | useRef | useState |
|------|--------|----------|
| 修改后重新渲染 | ❌ 不会 | ✅ 会 |
| 值保存在 | `.current` 属性上 | 直接返回 |
| 用途 | DOM引用、保存定时器等 | 页面上要展示的数据 |

### 4. useMemo — 缓存计算结果

> 避免每次渲染都重复做**昂贵的计算**。

```jsx
import { useMemo } from 'react';

function ProductList({ products, keyword }) {
  // 只在 products 或 keyword 变化时才重新过滤
  // 其他 state 变化导致的渲染不会重新计算
  const filtered = useMemo(() => {
    console.log('重新过滤...');
    return products.filter(p => p.name.includes(keyword));
  }, [products, keyword]);

  return (
    <ul>
      {filtered.map(p => <li key={p.id}>{p.name}</li>)}
    </ul>
  );
}
```

### 5. useCallback — 缓存函数

> 避免父组件每次渲染都创建**新的函数引用**，导致子组件不必要的重新渲染。

```jsx
import { useCallback, memo } from 'react';

// 子组件用 memo 包裹（props 不变就不重新渲染）
const Button = memo(({ onClick, text }) => {
  console.log('Button 渲染了');
  return <button onClick={onClick}>{text}</button>;
});

function Parent() {
  const [count, setCount] = useState(0);
  const [name, setName] = useState('张三');

  // ❌ 每次 Parent 渲染都会创建新函数 → Button 每次都重新渲染
  // const handleClick = () => setCount(c => c + 1);

  // ✅ 用 useCallback 缓存函数 → 依赖不变，函数引用不变 → Button 不重新渲染
  const handleClick = useCallback(() => {
    setCount(c => c + 1);
  }, []);  // 空依赖 = 函数永远不变

  return (
    <div>
      <p>{count} - {name}</p>
      <Button onClick={handleClick} text="加1" />
      <button onClick={() => setName('李四')}>改名</button>
      {/* 改名时 Button 不会重新渲染 */}
    </div>
  );
}
```

### 6. useContext — 跨组件传值

```jsx
import { createContext, useContext } from 'react';

const UserContext = createContext(null);

function App() {
  return (
    <UserContext.Provider value={{ name: '张三', role: 'admin' }}>
      <Dashboard />
    </UserContext.Provider>
  );
}

// 任意深层子组件直接取值
function UserInfo() {
  const { name, role } = useContext(UserContext);
  return <p>{name} - {role}</p>;
}
```

### 7. useReducer — 复杂状态管理

> 状态逻辑复杂时（多个相关的 state），用 useReducer 替代多个 useState。

```jsx
import { useReducer } from 'react';

// 定义 reducer（纯函数：根据 action 返回新 state）
function todoReducer(state, action) {
  switch (action.type) {
    case 'ADD':
      return [...state, { id: Date.now(), text: action.text, done: false }];
    case 'TOGGLE':
      return state.map(item =>
        item.id === action.id ? { ...item, done: !item.done } : item
      );
    case 'DELETE':
      return state.filter(item => item.id !== action.id);
    default:
      return state;
  }
}

function TodoList() {
  const [todos, dispatch] = useReducer(todoReducer, []);

  return (
    <div>
      <button onClick={() => dispatch({ type: 'ADD', text: '新任务' })}>
        添加
      </button>
      {todos.map(todo => (
        <div key={todo.id}>
          <span onClick={() => dispatch({ type: 'TOGGLE', id: todo.id })}>
            {todo.done ? '✅' : '⬜'} {todo.text}
          </span>
          <button onClick={() => dispatch({ type: 'DELETE', id: todo.id })}>
            删除
          </button>
        </div>
      ))}
    </div>
  );
}
```

## 三、自定义 Hook

> 以 `use` 开头的函数，把可复用的逻辑抽出来。

```jsx
// 自定义 Hook：监听窗口大小
function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  });

  useEffect(() => {
    const handleResize = () => {
      setSize({ width: window.innerWidth, height: window.innerHeight });
    };
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return size;
}

// 使用
function App() {
  const { width, height } = useWindowSize();
  return <p>窗口：{width} x {height}</p>;
}
```

```jsx
// 自定义 Hook：请求数据
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    fetch(url)
      .then(res => res.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}

// 使用
function UserPage() {
  const { data, loading, error } = useFetch('/api/user');
  if (loading) return <p>加载中...</p>;
  if (error) return <p>出错了</p>;
  return <p>{data.name}</p>;
}
```

## 四、Hooks 速查表

| Hook | 用途 | 简记 |
|------|------|------|
| `useState` | 状态管理 | 数据变了要重新渲染 |
| `useEffect` | 副作用（请求、定时器、订阅） | 渲染之后做的事 |
| `useRef` | DOM引用 / 保存不触发渲染的值 | .current 存值 |
| `useMemo` | 缓存计算结果 | 避免重复计算 |
| `useCallback` | 缓存函数引用 | 配合 memo 避免子组件重新渲染 |
| `useContext` | 跨组件取值 | 避免 props 层层传递 |
| `useReducer` | 复杂状态管理 | 多个相关 state 用它 |
| `自定义 Hook` | 复用逻辑 | use 开头的函数 |

## 五、高频面试题

### Q1：useMemo 和 useCallback 的区别？

```jsx
// useMemo 缓存的是 值（计算结果）
const value = useMemo(() => computeExpensiveValue(a, b), [a, b]);

// useCallback 缓存的是 函数本身
const fn = useCallback(() => doSomething(a), [a]);

// 其实 useCallback(fn, deps) 等价于 useMemo(() => fn, deps)
```

### Q2：useEffect 的清理函数什么时候执行？

1. 组件**卸载**时执行
2. 下次 effect **重新执行之前**执行（先清理上一次的）

### Q3：为什么 Hooks 不能放在条件语句里？

React 靠**调用顺序**来对应每个 Hook 的状态。如果放在 if 里，不同渲染可能调用顺序不同，React 就搞混了：

```jsx
// 第一次：useState → useState → useEffect ✅
// 第二次（if 不满足）：useState → useEffect ❌ 顺序对不上了！
```
