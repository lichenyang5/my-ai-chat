# React 18 新特性与原理

## 一、并发模式（Concurrent Mode）

> 一句话：让 React 能**同时准备多个版本的 UI**，高优先级更新可以打断低优先级更新。

```
React 17（同步渲染）：
用户输入 → ████████████ 渲染完 → 响应
                ↑ 这段时间页面卡住

React 18（并发渲染）：
用户输入 → ██ 暂停 → 响应输入 → ██ 继续渲染 → 完成
              ↑ 让出主线程，先响应用户
```

并发模式不是一个开关，而是一组新特性的基础。通过 `createRoot` 开启：

```jsx
// React 17
ReactDOM.render(<App />, document.getElementById('root'));

// React 18：开启并发特性
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
```

## 二、useTransition —— 标记低优先级更新

> 一句话：告诉 React "这个更新不急，用户交互优先"。

```jsx
import { useState, useTransition } from 'react';

function SearchPage() {
  const [keyword, setKeyword] = useState('');
  const [results, setResults] = useState([]);
  const [isPending, startTransition] = useTransition();

  const handleChange = (e) => {
    // 高优先级：输入框立即响应
    setKeyword(e.target.value);

    // 低优先级：搜索结果可以延后，不阻塞输入
    startTransition(() => {
      setResults(search(e.target.value));
    });
  };

  return (
    <div>
      <input value={keyword} onChange={handleChange} />
      {isPending && <span>搜索中...</span>}
      <ResultList data={results} />
    </div>
  );
}
```

## 三、useDeferredValue —— 延迟更新值

> 一句话：给一个值加"防抖"效果，让 React 在空闲时才更新它。

```jsx
import { useState, useDeferredValue } from 'react';

function SearchPage() {
  const [keyword, setKeyword] = useState('');
  // deferredKeyword 会延迟更新，不阻塞输入
  const deferredKeyword = useDeferredValue(keyword);

  return (
    <div>
      <input value={keyword} onChange={e => setKeyword(e.target.value)} />
      {/* 用 deferredKeyword 渲染列表，输入不卡 */}
      <HeavyList keyword={deferredKeyword} />
    </div>
  );
}
```

**useTransition vs useDeferredValue**：

| 对比 | useTransition | useDeferredValue |
|------|-------------|-----------------|
| 控制的是 | setState 调用 | 一个值 |
| 使用场景 | 你能控制 setState 的地方 | 你只能拿到值（比如 props） |
| 返回 | [isPending, startTransition] | 延迟后的值 |

## 四、自动批处理（Automatic Batching）

> React 17 只在事件处理函数中批处理，React 18 **所有场景**都自动批处理。

```jsx
// React 17：setTimeout 里的 setState 不会批处理 → 渲染 2 次
setTimeout(() => {
  setCount(1);   // 渲染一次
  setFlag(true); // 又渲染一次
}, 1000);

// React 18：自动批处理 → 只渲染 1 次
setTimeout(() => {
  setCount(1);   // ┐
  setFlag(true); // ┘ 合并，只渲染一次
}, 1000);

// 如果确实需要立即渲染（极少用）
import { flushSync } from 'react-dom';
flushSync(() => setCount(1));  // 立即渲染
flushSync(() => setFlag(true)); // 再渲染
```

## 五、Suspense 增强

React 18 的 Suspense 不再只是配合 `lazy` 做代码分割，还能配合数据请求：

```jsx
import { Suspense } from 'react';

// 配合支持 Suspense 的数据请求库（如 SWR、React Query、Relay）
function App() {
  return (
    <Suspense fallback={<Skeleton />}>
      <UserProfile />  {/* 数据没加载完显示 Skeleton */}
    </Suspense>
  );
}

// 嵌套 Suspense：不同区域独立加载
function Dashboard() {
  return (
    <div>
      <Suspense fallback={<HeaderSkeleton />}>
        <Header />
      </Suspense>
      <Suspense fallback={<ContentSkeleton />}>
        <Content />  {/* Header 加载完了 Content 还没好，各自独立 */}
      </Suspense>
    </div>
  );
}
```

## 六、Server Components（RSC）

> 一句话：组件在**服务端**运行，只把渲染结果发给客户端，JS bundle 更小。

```
传统 CSR：
服务端 → 发送 JS bundle → 客户端执行 JS → 渲染页面
                ↑ bundle 很大，首屏慢

Server Components：
服务端 → 执行组件，生成 HTML/数据 → 发送结果 → 客户端直接显示
                                      ↑ 不发送组件的 JS 代码
```

| 对比 | Client Component | Server Component |
|------|-----------------|-----------------|
| 运行在 | 浏览器 | 服务端 |
| 能用 state/effect | 能 | 不能 |
| 能访问数据库 | 不能 | 能 |
| 包含在 JS bundle | 是 | 否（零 JS） |
| 交互性 | 有 | 无（纯展示） |

> Next.js 13+ 的 App Router 默认使用 Server Components，需要交互的组件加 `'use client'`。

## 七、其他新增 Hooks

### useId —— 生成唯一 ID

```jsx
function FormField({ label }) {
  const id = useId();  // 服务端和客户端生成相同的 ID
  return (
    <>
      <label htmlFor={id}>{label}</label>
      <input id={id} />
    </>
  );
}
```

### useSyncExternalStore —— 订阅外部数据源

```jsx
import { useSyncExternalStore } from 'react';

function useWindowWidth() {
  return useSyncExternalStore(
    (callback) => {
      window.addEventListener('resize', callback);
      return () => window.removeEventListener('resize', callback);
    },
    () => window.innerWidth
  );
}
```

## 八、高频面试题

### Q1：React 18 有哪些重要更新？

核心是**并发渲染**。具体包括：createRoot API、自动批处理、useTransition、useDeferredValue、Suspense 增强、Server Components、useId 等。

### Q2：useTransition 和 useDeferredValue 怎么选？

- 你能控制 setState → 用 `useTransition`
- 你只能拿到一个值（比如 props 传来的）→ 用 `useDeferredValue`
- 两者本质都是标记低优先级更新

### Q3：什么是并发渲染？

React 可以同时准备多个版本的 UI，高优先级更新能打断低优先级更新。不是把渲染放到 setTimeout 里，而是把渲染工作**拆成小块**，在每一帧的空闲时间执行，保证用户交互不卡顿。

### Q4：自动批处理和之前有什么区别？

React 17 只在**事件处理函数**中批处理 setState。React 18 在所有场景（setTimeout、Promise、原生事件）中都自动批处理，减少不必要的渲染次数。

### Q5：Server Components 和 SSR 有什么区别？

| 对比 | SSR | Server Components |
|------|-----|------------------|
| 做什么 | 服务端生成 HTML，客户端 hydrate | 服务端运行组件，不发送 JS |
| JS bundle | 全量发送 | 只发送 Client Component 的 JS |
| 交互 | hydrate 后可交互 | Server Component 本身不可交互 |
| 数据请求 | getServerSideProps 等 | 组件内直接 async/await |
