# 核心模块：fs、Stream、process

## 一、fs 模块（文件系统）

`fs` 是 Node.js 内置的文件操作模块，不需要安装。

```js
const fs = require('fs');
const fsp = require('fs/promises'); // Promise 版本
```

### 常用 API 一览

| 操作 | 同步 | 异步（回调） | 异步（Promise） |
|------|------|-------------|----------------|
| 读文件 | `readFileSync` | `readFile` | `fsp.readFile` |
| 写文件 | `writeFileSync` | `writeFile` | `fsp.writeFile` |
| 追加内容 | `appendFileSync` | `appendFile` | `fsp.appendFile` |
| 删除文件 | `unlinkSync` | `unlink` | `fsp.unlink` |
| 创建目录 | `mkdirSync` | `mkdir` | `fsp.mkdir` |
| 读取目录 | `readdirSync` | `readdir` | `fsp.readdir` |
| 文件信息 | `statSync` | `stat` | `fsp.stat` |
| 重命名 | `renameSync` | `rename` | `fsp.rename` |
| 文件是否存在 | `existsSync` | — | `fsp.access` |

### 基本用法

```js
// ✅ 推荐：Promise 方式（async/await）
const fs = require('fs/promises');

// 读文件
const data = await fs.readFile('./test.txt', 'utf-8');

// 写文件（覆盖）
await fs.writeFile('./test.txt', 'Hello Node');

// 追加内容
await fs.appendFile('./test.txt', '\n新的一行');

// 创建目录（recursive: true 递归创建）
await fs.mkdir('./a/b/c', { recursive: true });

// 获取文件信息
const stat = await fs.stat('./test.txt');
console.log(stat.isFile());      // true
console.log(stat.isDirectory()); // false
console.log(stat.size);          // 文件大小（字节）
```

### 面试题：同步 vs 异步怎么选？

- **异步优先**：不阻塞事件循环，适合服务端场景
- **同步场景**：程序初始化阶段读配置文件，可以用同步（只执行一次）

## 二、Stream（流）

### Stream 是什么？

> 一句话：**分块处理数据**，不用一次性把整个文件加载到内存。

类比：你看视频不用等全部下载完才看，边下载边播放——这就是流。

### 四种流类型

| 类型 | 作用 | 示例 |
|------|------|------|
| **Readable** | 可读流 | `fs.createReadStream`、`http.request` |
| **Writable** | 可写流 | `fs.createWriteStream`、`http.response` |
| **Duplex** | 双工流（可读可写） | `net.Socket`、`WebSocket` |
| **Transform** | 转换流（读入→变换→写出） | `zlib.createGzip`（压缩） |

### 基本用法

```js
const fs = require('fs');

// 读取流
const readStream = fs.createReadStream('./big-file.txt', {
  encoding: 'utf-8',
  highWaterMark: 64 * 1024  // 每次读 64KB
});

readStream.on('data', (chunk) => {
  console.log('收到一块数据:', chunk.length);
});
readStream.on('end', () => {
  console.log('读取完毕');
});
readStream.on('error', (err) => {
  console.error('读取出错:', err);
});

// 写入流
const writeStream = fs.createWriteStream('./output.txt');
writeStream.write('第一行\n');
writeStream.write('第二行\n');
writeStream.end('最后一行');  // end 后不能再 write

// ✅ 最常用：pipe（管道）—— 把读取流直接接到写入流
fs.createReadStream('./big-file.txt')
  .pipe(fs.createWriteStream('./copy.txt'));
```

### 为什么用 Stream？

```js
// ❌ 不用流：一次性读入内存（大文件直接炸）
const data = fs.readFileSync('./2GB-file.txt');

// ✅ 用流：分块处理，内存占用稳定
fs.createReadStream('./2GB-file.txt')
  .pipe(fs.createWriteStream('./copy.txt'));
```

| 方式 | 内存占用 | 适合场景 |
|------|---------|---------|
| `readFile` | 整个文件 | 小文件（< 几MB） |
| `createReadStream` | 一小块（64KB） | 大文件、视频、日志 |

## 三、process（进程）

`process` 是一个**全局对象**，不需要 require，提供当前 Node.js 进程的信息和控制。

### 常用属性

```js
// 环境变量
process.env.NODE_ENV          // 'development' | 'production'
process.env.PORT              // 自定义环境变量

// 命令行参数
process.argv                  // ['node路径', '脚本路径', '参数1', '参数2']
// 例：node app.js --port 3000
// process.argv = ['/usr/bin/node', '/app.js', '--port', '3000']

// 进程信息
process.pid                   // 进程 ID
process.ppid                  // 父进程 ID
process.platform              // 'darwin'(Mac) | 'win32' | 'linux'
process.version               // Node.js 版本
process.cwd()                 // 当前工作目录
process.memoryUsage()         // 内存使用情况
process.uptime()              // 进程运行时间（秒）
```

### 常用方法

```js
// 退出进程
process.exit(0);              // 0=正常退出，1=异常退出

// 下一个微任务（比 Promise.then 更早执行）
process.nextTick(() => {
  console.log('在当前阶段结束后、下个阶段开始前执行');
});

// 标准输入输出
process.stdout.write('输出');   // 等同于 console.log 但不换行
process.stdin.on('data', (data) => {
  console.log('用户输入:', data.toString());
});
```

### 进程事件

```js
// 未捕获异常（兜底，防止进程崩溃）
process.on('uncaughtException', (err) => {
  console.error('未捕获异常:', err);
  process.exit(1);  // 建议记录日志后退出
});

// 未处理的 Promise 拒绝
process.on('unhandledRejection', (reason) => {
  console.error('未处理的Promise拒绝:', reason);
});

// 进程退出前（只能做同步操作）
process.on('exit', (code) => {
  console.log('进程即将退出，退出码:', code);
});

// 接收信号（优雅关闭）
process.on('SIGTERM', () => {
  console.log('收到终止信号，开始优雅关闭...');
  server.close(() => process.exit(0));
});
```

### 面试题：process.nextTick 和 setImmediate 的区别？

| 对比 | `process.nextTick` | `setImmediate` |
|------|-------------------|----------------|
| 执行时机 | 当前阶段结束后立即执行 | 在 check 阶段执行 |
| 优先级 | 更高 | 较低 |
| 队列 | 微任务队列 | 宏任务队列 |
| 风险 | 递归调用会阻塞事件循环 | 不会阻塞 |

```js
setImmediate(() => console.log('setImmediate'));
process.nextTick(() => console.log('nextTick'));
// 输出：nextTick → setImmediate
```
