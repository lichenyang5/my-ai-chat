# 文件上传

## 一、文件上传原理

浏览器通过 `<input type="file">` 选择文件，用 **`multipart/form-data`** 格式发送给服务器。

```
普通表单：Content-Type: application/x-www-form-urlencoded
文件上传：Content-Type: multipart/form-data; boundary=----WebKitFormBoundary
```

> `multipart/form-data` 会把文件拆成多个"部分"，每部分有独立的 Header，用 boundary 分隔。

### 请求报文长什么样？

```
POST /upload HTTP/1.1
Content-Type: multipart/form-data; boundary=----abc123

------abc123
Content-Disposition: form-data; name="username"

张三
------abc123
Content-Disposition: form-data; name="avatar"; filename="photo.jpg"
Content-Type: image/jpeg

（二进制文件内容...）
------abc123--
```

## 二、Node.js 实现文件上传

### 方案一：Multer（Express 常用）

**Multer 是什么？**
Multer 是一个专门处理 `multipart/form-data` 的 Express 中间件，用于处理文件上传。它会自动解析请求中的文件数据，保存到磁盘或内存，并把文件信息挂到 `req.file`（单文件）或 `req.files`（多文件）上，开发者不需要手动解析二进制数据。

```bash
npm install multer
```

```js
const express = require('express');
const multer = require('multer');  // 文件上传中间件，处理 multipart/form-data
const path = require('path');

const app = express();

// ========== 第一步：配置存储策略 ==========
// diskStorage 表示文件存到磁盘（另一个选项是 memoryStorage 存到内存）
const storage = multer.diskStorage({
  // destination: 文件存到哪个目录
  // cb 是回调函数，第一个参数是错误（null 表示没错），第二个是目标路径
  destination: (req, file, cb) => {
    cb(null, 'uploads/');  // 存到项目根目录的 uploads/ 文件夹
  },
  // filename: 存储时用什么文件名（不设置的话 multer 会生成随机文件名，没有扩展名）
  filename: (req, file, cb) => {
    const ext = path.extname(file.originalname);  // 获取原始文件扩展名，如 .jpg
    cb(null, Date.now() + ext);  // 最终文件名如：1710000000000.jpg
  }
});

// ========== 第二步：配置文件过滤（可选） ==========
// 决定哪些文件可以上传，哪些被拒绝
const fileFilter = (req, file, cb) => {
  // 允许：图片、Word 文档、Markdown 文件
  const allowTypes = [
    'image/jpeg', 'image/png', 'image/gif',             // 图片
    'application/msword',                                 // .doc
    'application/vnd.openxmlformats-officedocument.wordprocessingml.document', // .docx
    'text/markdown', 'text/x-markdown',                   // .md
  ];
  if (allowTypes.includes(file.mimetype)) {
    cb(null, true);   // true = 接受该文件
  } else {
    cb(new Error('只允许上传图片、Word 或 Markdown 文件'), false);
  }
};

// ========== 第三步：创建 multer 实例 ==========
const upload = multer({
  storage,      // 存储策略（上面定义的）
  fileFilter,   // 文件过滤器（上面定义的）
  limits: { fileSize: 5 * 1024 * 1024 }  // 限制文件大小最大 5MB
});

// ========== 第四步：在路由中使用 ==========

// 单文件上传
// upload.single('file') 表示接收 name="file" 的单个文件字段
// 上传后文件信息在 req.file 中
app.post('/upload', upload.single('file'), (req, res) => {
  // req.file 包含：fieldname, originalname, filename, path, size, mimetype 等
  res.json({
    message: '上传成功',
    filename: req.file.filename,  // 存储后的文件名
    originalname: req.file.originalname, // 原始文件名
    size: req.file.size           // 文件大小（字节）
  });
});

// 多文件上传
// upload.array('files', 5) 表示接收 name="files" 的文件字段，最多 5 个
// 上传后文件信息在 req.files 数组中
app.post('/uploads', upload.array('files', 5), (req, res) => {
  res.json({
    message: '上传成功',
    files: req.files.map(f => ({
      filename: f.filename,
      originalname: f.originalname,
      size: f.size
    }))
  });
});
```

**Multer 常用方法总结**：

| 方法 | 作用 | 文件信息位置 |
|------|------|------------|
| `upload.single('fieldName')` | 接收单个文件 | `req.file` |
| `upload.array('fieldName', maxCount)` | 接收多个同名文件 | `req.files` |
| `upload.fields([{name: 'a'}, {name: 'b'}])` | 接收多个不同名文件 | `req.files.a` / `req.files.b` |
| `upload.none()` | 只接收文本字段，不接收文件 | — |

## 三、前端 React 上传组件

### 基础上传组件（图片/Word/Markdown）

```jsx
import { useState, useRef } from 'react';
import axios from 'axios';  // npm install axios

// 允许上传的文件类型
const ACCEPT_TYPES = {
  'image/*': '图片',
  '.doc,.docx': 'Word 文档',
  '.md': 'Markdown 文件',
};
// 拼成 accept 属性的字符串：image/*,.doc,.docx,.md
const ACCEPT_STRING = Object.keys(ACCEPT_TYPES).join(',');

export default function FileUpload() {
  const [fileInfo, setFileInfo] = useState(null);   // 上传成功后的文件信息
  const [preview, setPreview] = useState(null);      // 图片预览 URL
  const [progress, setProgress] = useState(0);       // 上传进度 0-100
  const [uploading, setUploading] = useState(false);  // 是否正在上传
  const [error, setError] = useState('');             // 错误信息
  const inputRef = useRef(null);                      // 引用 <input> 元素

  // ========== 选择文件后触发 ==========
  const handleFileChange = (e) => {
    const file = e.target.files[0];
    if (!file) return;

    setError('');
    setFileInfo(null);

    // 如果是图片，生成预览
    if (file.type.startsWith('image/')) {
      const url = URL.createObjectURL(file);  // 创建临时预览 URL
      setPreview(url);
    } else {
      setPreview(null);
    }

    // 自动上传
    uploadFile(file);
  };

  // ========== 上传文件 ==========
  const uploadFile = async (file) => {
    setUploading(true);
    setProgress(0);

    // 1. 用 FormData 包装文件
    const formData = new FormData();
    formData.append('file', file);  // 'file' 要和后端 upload.single('file') 一致

    try {
      // 2. 用 axios 上传（自带进度监听，比 xhr 简洁）
      const { data } = await axios.post('/upload', formData, {
        // 不需要手动设置 Content-Type，axios 检测到 FormData 会自动加
        // 3. onUploadProgress 监听上传进度
        onUploadProgress: (e) => {
          const percent = Math.round((e.loaded / e.total) * 100);
          setProgress(percent);
        },
      });
      setFileInfo(data);
    } catch (err) {
      setError(err.response?.data?.message || '上传失败');
    } finally {
      setUploading(false);
    }
  };

  return (
    <div>
      <h3>文件上传</h3>

      {/* accept 限制文件选择器只显示允许的类型 */}
      <input
        ref={inputRef}
        type="file"
        accept={ACCEPT_STRING}
        onChange={handleFileChange}
        style={{ display: 'none' }}  {/* 隐藏原生 input */}
      />

      {/* 自定义上传按钮 */}
      <button onClick={() => inputRef.current.click()} disabled={uploading}>
        {uploading ? '上传中...' : '选择文件（图片/Word/MD）'}
      </button>

      {/* 上传进度条 */}
      {uploading && (
        <div style={{ width: 300, background: '#eee', marginTop: 10 }}>
          <div style={{
            width: `${progress}%`,
            height: 8,
            background: '#4caf50',
            transition: 'width 0.3s'
          }} />
          <span>{progress}%</span>
        </div>
      )}

      {/* 图片预览 */}
      {preview && <img src={preview} alt="预览" style={{ maxWidth: 300, marginTop: 10 }} />}

      {/* 上传结果 */}
      {fileInfo && (
        <div style={{ marginTop: 10, color: 'green' }}>
          ✅ 上传成功：{fileInfo.originalname}（{(fileInfo.size / 1024).toFixed(1)} KB）
        </div>
      )}

      {/* 错误提示 */}
      {error && <div style={{ marginTop: 10, color: 'red' }}>❌ {error}</div>}
    </div>
  );
}
```

### 不需要进度条时的简化写法

```jsx
const uploadFile = async (file) => {
  const formData = new FormData();
  formData.append('file', file);

  try {
    const { data } = await axios.post('/upload', formData);
    // axios 传 FormData 时会自动设置 Content-Type，不需要手动加
    console.log('上传成功:', data);
  } catch (err) {
    console.error('上传失败:', err.response?.data || err.message);
  }
};
```

### 多文件上传

```jsx
// input 加 multiple 属性
<input type="file" accept={ACCEPT_STRING} multiple onChange={handleMultiUpload} />

const handleMultiUpload = (e) => {
  const files = Array.from(e.target.files);  // FileList 转数组
  const formData = new FormData();

  // 所有文件用同一个字段名，和后端 upload.array('files', 5) 对应
  files.forEach((file) => {
    formData.append('files', file);
  });

  axios.post('/uploads', formData)
    .then(({ data }) => console.log(data))
    .catch((err) => console.error(err));
};
```

### 关键点总结

| 要点 | 说明 |
|------|------|
| **FormData** | 用来包装文件，`append('字段名', file)` 字段名要和后端一致 |
| **Content-Type** | 不要手动设置！浏览器会自动加 `multipart/form-data; boundary=xxx` |
| **accept 属性** | 只是限制文件选择器的显示，**不能防止用户绕过**，后端一定要二次校验 |
| **进度监听** | axios 用 `onUploadProgress` 回调即可，比 XHR 原生写法简洁 |
| **图片预览** | `URL.createObjectURL(file)` 创建临时 URL，不需要先上传到服务器 |

## 四、大文件上传（分片上传）

大文件一次上传容易超时或失败，解决方案：**切片上传 + 合并**。

### 原理

```
大文件（100MB）
    ↓ 前端切片
[片1: 0-5MB] [片2: 5-10MB] ... [片20: 95-100MB]
    ↓ 逐个/并发上传
服务端接收所有切片
    ↓ 合并
完整文件
```

### 前端切片

```js
function sliceFile(file, chunkSize = 5 * 1024 * 1024) {
  const chunks = [];
  let start = 0;

  while (start < file.size) {
    chunks.push(file.slice(start, start + chunkSize));
    start += chunkSize;
  }

  return chunks;
}

// 上传所有切片
async function uploadChunks(file) {
  const chunks = sliceFile(file);
  const hash = await calculateHash(file);  // 用 MD5/SparkMD5 算文件哈希

  // 并发上传
  await Promise.all(chunks.map((chunk, index) => {
    const formData = new FormData();
    formData.append('chunk', chunk);
    formData.append('hash', hash);
    formData.append('index', index);
    formData.append('total', chunks.length);
    return fetch('/upload/chunk', { method: 'POST', body: formData });
  }));

  // 通知服务端合并
  await fetch('/upload/merge', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ hash, filename: file.name, total: chunks.length })
  });
}
```

### 服务端合并

```js
const fs = require('fs');
const path = require('path');

app.post('/upload/merge', async (req, res) => {
  const { hash, filename, total } = req.body;
  const chunkDir = path.join('uploads/chunks', hash);
  const outputPath = path.join('uploads', filename);

  // 按顺序合并
  const writeStream = fs.createWriteStream(outputPath);
  for (let i = 0; i < total; i++) {
    const chunkPath = path.join(chunkDir, String(i));
    const data = fs.readFileSync(chunkPath);
    writeStream.write(data);
    fs.unlinkSync(chunkPath);  // 删除切片
  }
  writeStream.end();

  fs.rmdirSync(chunkDir);
  res.json({ message: '合并成功', path: outputPath });
});
```

## 五、断点续传

在分片上传基础上，记录已上传的切片，下次从断点继续。

```js
// 上传前先问服务端：哪些切片已经传过了？
app.get('/upload/progress/:hash', (req, res) => {
  const chunkDir = path.join('uploads/chunks', req.params.hash);
  if (!fs.existsSync(chunkDir)) {
    return res.json({ uploaded: [] });
  }
  const uploaded = fs.readdirSync(chunkDir).map(Number);
  res.json({ uploaded });
});

// 前端：跳过已上传的切片
```

## 六、高频面试题

### Q1：前端上传文件为什么不能手动设置 Content-Type？

因为 `multipart/form-data` 需要带 **boundary**（分隔符），浏览器会自动生成。手动设置就没有 boundary，服务器解析不了。

### Q2：大文件上传的完整方案？

1. 前端用 `file.slice()` 切片
2. 用 SparkMD5 算文件哈希（秒传判断 + 标识文件）
3. 并发上传切片
4. 服务端收齐后合并
5. 支持断点续传（记录已上传的切片序号）

### Q3：如何实现秒传？

上传前先算文件 MD5 哈希，发给服务端查询：
- 如果服务端已有相同哈希的文件 → 直接返回"上传成功"（秒传）
- 如果没有 → 走正常上传流程
