# 中间件机制

## 一、中间件是什么？

> 一句话：**请求和响应之间的"流水线工人"**，每个中间件处理一部分工作，然后交给下一个。

```
请求 → [中间件1] → [中间件2] → [中间件3] → 路由处理 → 响应
         日志        鉴权        解析Body
```

类比：快递从发出到收货，经过分拣中心、运输、派件——每个环节就是一个"中间件"。

## 二、Express 中间件

### 基本写法

```js
// 中间件就是一个函数，接收 req、res、next 三个参数
function myMiddleware(req, res, next) {
  console.log('我是中间件，做点事情...');
  next();  // ← 调用 next() 才会继续往下走，不调就卡住了
}

app.use(myMiddleware);
```

### 中间件分类

| 类型 | 用法 | 说明 |
|------|------|------|
| **应用级** | `app.use(fn)` | 对所有请求生效 |
| **路由级** | `router.use(fn)` | 只对某个路由组生效 |
| **错误处理** | `app.use((err, req, res, next) => {})` | 4个参数，专门处理错误 |
| **内置** | `express.json()`、`express.static()` | Express 自带的 |
| **第三方** | `cors()`、`morgan()` | npm 安装的 |

### 实际例子

```js
const express = require('express');
const app = express();

// 1. 日志中间件（应用级）
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url} - ${new Date().toISOString()}`);
  next();
});

// 2. 解析 JSON 请求体（内置中间件）
app.use(express.json());

// 3. 路由
app.get('/api/users', (req, res) => {
  res.json({ users: [] });
});

// 4. 错误处理中间件（放在最后）
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ message: '服务器内部错误' });
});
```

## 三、Koa 中间件（洋葱模型）

Koa 的中间件采用**洋葱模型**——请求从外到内穿过每一层，响应再从内到外穿回来。

```
        请求 ──────────→
    ┌─────────────────────────┐
    │ 中间件1（前半段）          │
    │  ┌──────────────────┐   │
    │  │ 中间件2（前半段）   │   │
    │  │  ┌───────────┐   │   │
    │  │  │  核心处理   │   │   │
    │  │  └───────────┘   │   │
    │  │ 中间件2（后半段）   │   │
    │  └──────────────────┘   │
    │ 中间件1（后半段）          │
    └─────────────────────────┘
        ←────────── 响应
```

### Koa 洋葱模型代码

```js
const Koa = require('koa');
const app = new Koa();

app.use(async (ctx, next) => {
  console.log('1 - 进入');       // 第1步
  await next();
  console.log('1 - 离开');       // 第6步
});

app.use(async (ctx, next) => {
  console.log('2 - 进入');       // 第2步
  await next();
  console.log('2 - 离开');       // 第5步
});

app.use(async (ctx, next) => {
  console.log('3 - 核心处理');    // 第3步
  ctx.body = 'Hello';           // 第4步
});

// 输出顺序：1进 → 2进 → 3核心 → 2出 → 1出
```

## 四、手写简易中间件机制

### Express 风格（线性）

```js
function createApp() {
  const middlewares = [];

  return {
    use(fn) {
      middlewares.push(fn);
    },
    run(req, res) {
      let index = 0;
      function next() {
        if (index < middlewares.length) {
          const fn = middlewares[index++];
          fn(req, res, next);
        }
      }
      next();
    }
  };
}
```

### Koa 风格（洋葱模型 / compose）

```js
function compose(middlewares) {
  return function (ctx) {
    let index = -1;

    function dispatch(i) {
      if (i <= index) return Promise.reject(new Error('next() 被多次调用'));
      index = i;
      const fn = middlewares[i];
      if (!fn) return Promise.resolve();
      return Promise.resolve(fn(ctx, () => dispatch(i + 1)));
    }

    return dispatch(0);
  };
}
```

> 这就是 `koa-compose` 的核心原理，面试常考手写。

## 五、高频面试题

### Q1：Express 和 Koa 中间件的区别？

| 对比 | Express | Koa |
|------|---------|-----|
| 模型 | 线性模型 | 洋葱模型 |
| 写法 | 回调函数 `(req, res, next)` | async 函数 `(ctx, next)` |
| 错误处理 | 需要特殊的 4 参数中间件 | 用 try/catch 包裹即可 |
| next | 同步调用 | `await next()`，支持等待后续执行完 |

### Q2：中间件的执行顺序？

- 按照 `app.use()` 的注册顺序执行
- 调用 `next()` 进入下一个中间件
- 不调用 `next()` 请求就会挂起（卡住）

### Q3：常见的中间件有哪些？

| 中间件 | 作用 |
|--------|------|
| `body-parser` / `express.json()` | 解析请求体 |
| `cors` | 处理跨域 |
| `morgan` | HTTP 请求日志 |
| `helmet` | 安全头部 |
| `compression` | Gzip 压缩 |
| `express-rate-limit` | 限流 |
| `multer` | 文件上传 |
