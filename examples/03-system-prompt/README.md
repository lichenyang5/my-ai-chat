# 示例03：系统提示词（System Prompt）

## 学习目标

理解系统提示词（System Prompt）：
1. 什么是系统提示词
2. 如何用它控制AI的行为
3. 为什么它是AI应用的核心

## 核心概念

### 1. 什么是系统提示词？

**System Prompt** = 给AI设定"人设"和规则

```typescript
await client.messages.create({
  model: "claude-3-5-sonnet-20241022",
  system: "你是一位专业的Python教师", // 👈 这就是系统提示词
  messages: [{ role: "user", content: "如何学习Python?" }],
});
```

### 2. System Prompt vs User Message

```
System Prompt（系统提示词）：
- 设定AI的角色、风格、规则
- 用户看不到（在背后生效）
- 整个对话期间保持不变
- 优先级最高

User Message（用户消息）：
- 用户的具体问题
- 对话的主体内容
- 每次可以不同
```

### 3. 好的 System Prompt 包含什么？

**基本结构：**
```typescript
const systemPrompt = `
你是一位[角色定位]。

## 你的专长
- [专业领域1]
- [专业领域2]

## 回答风格
- [风格要求1]
- [风格要求2]

## 注意事项
- [限制条件1]
- [限制条件2]
`;
```

**实际例子：**
```typescript
const systemPrompt = `
你是一位资深的JavaScript导师，有10年教学经验。

## 你的专长
- 前端开发（React、Vue）
- Node.js后端开发
- TypeScript

## 回答风格
- 先给简短答案，再详细解释
- 总是提供可运行的代码示例
- 用简单类比帮助理解

## 注意事项
- 不要使用过时的语法（如var）
- 总是使用TypeScript类型注解
- 提醒常见陷阱和最佳实践
`;
```

### 4. System Prompt 的威力

**同样的问题，不同的System Prompt：**

问题：`"什么是递归？"`

**System Prompt 1**: "你是专业教师"
回答：递归是函数调用自身的编程技术...（详细教学）

**System Prompt 2**: "你是海盗船长"
回答：哈哈！递归就像俺的船队，大船里有小船，小船里还有更小的船...（海盗风格）

**System Prompt 3**: "你是资深程序员"
回答：简单说，函数调自己。示例代码：`function factorial(n) { ... }`

## 运行示例

```bash
pnpm example:03
```

## 预期输出

你会看到同一个问题（"什么是递归？"）在不同的系统提示词下：
- 默认AI助手：标准解释
- 专业技术顾问：简洁准确
- 耐心的老师：通俗易懂，带比喻
- 海盗船长：海盗腔调（有趣！）
- 资深程序员：代码示例为主

## 实际应用场景

### 场景1：客服机器人
```typescript
system: `
你是某电商平台的客服助手。
- 友好、礼貌、有耐心
- 先理解客户问题，再提供解决方案
- 如果无法解决，引导联系人工客服
- 不承诺超出权限的事情
`;
```

### 场景2：代码审查助手
```typescript
system: `
你是代码审查专家。
审查时关注：
1. 代码规范和风格
2. 潜在的bug和边界情况
3. 性能问题
4. 安全漏洞
用checklist形式给出建议。
`;
```

### 场景3：内容创作助手
```typescript
system: `
你是小红书文案专家。
风格要求：
- 轻松活泼，使用emoji
- 短句为主，易读性强
- 适当的口语化表达
- 包含3-5个相关话题标签
`;
```

## Prompt Engineering 技巧

### 1. 清晰的角色定位
❌ "你是AI助手"（太笼统）
✅ "你是专注于React性能优化的前端专家"（具体）

### 2. 明确的输出格式
```typescript
system: `
回答格式：
1. 一句话总结（不超过20字）
2. 详细解释（3-5段）
3. 相关资源链接
`;
```

### 3. 示例引导（Few-shot）
```typescript
system: `
你是翻译助手。

示例：
输入：Hello
输出：你好

输入：Thank you
输出：谢谢

现在开始翻译用户输入。
`;
```

### 4. 限制和边界
```typescript
system: `
注意：
- 不回答医疗诊断问题
- 不提供投资建议
- 遇到这些问题，礼貌地拒绝并说明原因
`;
```

## 调试 Prompt 的方法

1. **对比测试**：同样的问题测试不同的prompt
2. **边界测试**：故意问一些边界问题，看AI如何处理
3. **迭代优化**：根据实际效果不断调整
